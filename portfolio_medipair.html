01 · Background

기획 배경 &
서비스 핵심 가치

약물-음식 상호작용 정보는 전문 서적에 흩어져 있어 일반인이 쉽게 확인하기 어렵습니다. 특히 고령자와 다약제 복용자의 오남용 위험이 증가하는 상황에서, 누구나 즉시 확인할 수 있는 AI 안전 가이드가 필요했습니다.

🔍

약물-음식 상호작용 정보 분산

정보가 전문 의약 서적·처방전에 흩어져 있어 일반인이 즉시 확인하기 어려움

👴

고령자·다약제 복용자 오남용 위험

여러 약을 동시에 복용하는 환자군에서 상호작용 부작용 위험 급증

📚

전문성 중심 정보의 낮은 접근성

의학 용어 중심 안내로 일반 사용자가 이해하기 어려운 구조

💡

직관적 AI 안전 가이드 부재

사진 한 장으로 즉시 위험도를 판단할 수 있는 서비스가 없음

📸 이미지 기반 즉시 안전 확인

알약·음식 사진만으로 상호작용 위험을 즉시 안내합니다. 텍스트 입력 없이 카메라만으로 분석 가능합니다.

✅ 누구나 이해하는 직관적 안전 가이드

Safe / Caution 2단계 위험도 분류로 의학 지식 없이도 즉시 이해할 수 있는 인터페이스를 설계했습니다.

🏥 정보 소외 계층 접근성 강화

고령자·디지털 취약 계층도 쉽게 사용할 수 있도록 복잡한 입력 없이 이미지만으로 작동하는 UX를 구현했습니다.

02 · Core Features

4가지
핵심 기능

알약 인식·음식 탐지·상호작용 DB 구축·결과 시각화. 4가지 기능이 파이프라인으로 연결되어 사진 한 장으로 안전 여부를 즉시 제공합니다.

1
💊

알약 인식 (Pill Recognition)

Azure Custom Vision Cognitive Services로 다양한 모양·색상의 알약을 정확히 식별합니다. 제형 특징 학습으로 분류 모델을 최적화했습니다.

Azure Custom Vision 이미지 분류 93% 정확도
2
🍽

음식 인식 (Food Detection)

OpenCV + YOLOv11 기반 실시간 한국 음식 객체 탐지. 한국 음식 데이터 기반으로 다중 객체 인식이 가능하며 사용자가 섭취하려는 음식을 자동 탐지·분류합니다.

YOLOv11 OpenCV 실시간 탐지
3
🗄

상호작용 DB 구축

공공 데이터를 활용해 약물·음식 위험도를 Safe / Caution 2단계로 체계적으로 분류했습니다. Azure SQL과 Open Data를 결합한 전문 의약 정보 시각화를 구현했습니다.

Safe / Caution 분류 Azure SQL 공공 데이터
4
📊

결과 시각화 & 안내

약물-음식 상호작용 분석 결과를 직관적 UI로 즉시 표시합니다. 주의 성분 명시, 복약 안내 메시지, 다른 조합 확인 기능으로 사용 편의성을 극대화했습니다.

직관적 UI/UX 주의 성분 명시 복약 안내
03 · Performance & Architecture

기술 성과 &
아키텍처 Deep Dive

이미지 전처리 파이프라인 강화와 데이터 불균형 해결로 93% 인식 정확도를 달성했습니다. 9,000+ 학습 데이터셋을 직접 구축했습니다.

ACHIEVEMENT 01

알약 인식 정확도 93% 달성

다양한 모양·색상의 알약을 카메라로 정확히 분류하는 것이 핵심 기술 과제였습니다. 조명 환경, 촬영 각도, 유사한 제형 간 구분이 특히 어려웠습니다.

해결 전략

이미지 전처리 파이프라인 강화(노이즈 제거·대비 조정·리사이징)와 Azure Custom Vision 모델 최적화로 제형 특징을 정밀 학습했습니다.

알약 인식 정확도 93% 달성
ACHIEVEMENT 02

9,000+ 대규모 데이터셋 구축

한국 음식 및 주요 처방 약물 이미지 데이터가 부족해 모델 학습에 필요한 양질의 데이터셋을 직접 구축해야 했습니다.

해결 전략

Food 6,000+ / Drug 3,000+ 이미지 데이터를 직접 수집·라벨링했습니다. 특정 클래스 데이터 부족 문제는 오버샘플링(Oversampling) 기법으로 해결했습니다.

총 9,000+ 학습 데이터 · 인식률 편차 최소화
ACHIEVEMENT 03

데이터 불균형 문제 해결

특정 클래스(희귀 약물, 특정 음식)의 데이터가 다른 클래스에 비해 현저히 부족해 모델이 편향 학습되는 문제가 발생했습니다.

해결 전략

오버샘플링 기법을 적용해 클래스 간 데이터 균형을 맞추고, 데이터 증강(Augmentation)으로 다양한 환경에서의 인식률을 향상했습니다.

인식률 편차 최소화 · 전체 클래스 균등 성능
